写在最前：愿你不需要这份指南

发表于 2025-05-20 分类于 游戏服务器架构与实现 阅读次数：

本系列将试图不那么不靠谱地介绍游戏后端从 0 到可上线的全过程。供参考，或以防万一。

写在最前：愿你不需要这份指南

这是一份什么样的指南？

它不会告诉你“怎么写个 HTTP 服务”，那类资料早已泛滥。

它也不会告诉你“用哪个框架最牛逼”，因为真正靠谱的框架，往往是在你们项目里，被你亲手调成合适的。

这份指南会尝试解答这些问题：

  • 游戏服务器到底是做什么的？为什么和 Web Server 差异巨大？
  • 市面上有哪些服务器架构，它们的演化路径是什么？
  • 一套多人在线游戏，服务器怎么搭、怎么扩容、怎么监控？
  • 每个核心系统（AOI、技能、背包、AI 等）在真实项目里应该怎么设计？
  • 如何处理日志、BI、线上调试、容灾和崩溃分析？

面向谁？

这套内容面向：

  • 想要搭建游戏服务器的独立开发者
  • 想要系统了解游戏后端的架构师、服务端工程师
  • 对 MMO、RPG、战斗类游戏后端有兴趣的在职/转岗程序员
  • 已在游戏项目中工作，正在尝试“把代码写得更好”的你
  • 和想要笔记下可行性经验的我~

不面向谁？

如果你正在寻找：

  • 如何部署 MySQL 或 Redis；
  • 如何用某款后端引擎（如 Photon）上手一小时做出游戏；
  • 如何使用 ECS 或 Kubernetes 管理实例；

这些会在其他更专门的文章中获得更好的解答。

为什么要写这份指南？

因为我在第一次搭建完整服务器架构的时候，查不到任何系统性的中文资料。

能找到的是：

  • 零散的论坛讨论
  • 工程师转述的线下分享

写下这份指南，是为了将这些“踩坑后的经验”系统整理出来：

  • 帮助未来的我，和未来的你
  • 如果能节省哪怕你 1 天的 debug 时间，写它就值了

写在最后
